Not a to-do list. A structure for holding nine real commitments at once —
Scripture, prayer, health, two organizations, learning, rest, and people —
without any one of them quietly eating another.

- **Day 1:** Monday, Jul 20, 2026

- **Buffer:** Oct 18 – Oct 31 — no new pushes, just finish anything unfinished,
  run the 90-day review, and set the next quarter. Don't skip this — it's
  where the gains actually get locked in.

## How the week is shaped
See `schedule.md` for the full breakdown. Short version:
- **Mon, Tue, Wed, Fri** — full work days
- **Sat** — work day, but includes Sunday prep (pastoral)
- **Sun** — church + pastoral work, no business work
- **Thu** — Sabbath: full stop. Rest day, and also the one exercise-off day.

## How to use this with Claude Code
1. Open this folder as a Claude Code project.
2. `CLAUDE.md` tells Claude where everything lives and how to run reviews.
3. Each night, fill `daily-log-template.md` — copy its contents into a new
   dated file named `daily-log-YYYY-MM-DD.md`, don't edit the template
   itself. Takes under 5 minutes.
4. Say "run my weekly review" on Sundays, and "run my monthly review" at the
   end of each ~30-day block. Claude will read the daily logs and walk you
   through `weekly-review-template.md` / `monthly-review-template.md`.

## Files
All files live flat in this project (no subfolders) — dated logs and
reviews are distinguished by filename, not by folder.
- `goals.md` — the 9 outcomes, each with a real metric and target
- `schedule.md` — day-type templates + the deep-work / engagement problem
- `bible.md`, `fitness.md`, `prayer.md`, `business.md`, `learning.md` —
  ongoing trackers for each goal
- `daily-log-template.md` — copy nightly into `daily-log-YYYY-MM-DD.md`
- `weekly-review-template.md` — copy weekly into `weekly-review-week-N-YYYY-MM-DD.md`
- `monthly-review-template.md` — copy monthly into `monthly-review-YYYY-MM.md`
- `food-notes.md` — organic/whole-food sourcing in the GTA
